# Warsaw Skate App

Flutter-прототип приложения для iPhone и Android, в котором:

- есть карта Варшавы и ближайших городов
- показываются типы покрытия разными цветами: асфальт, плитка, велодорожка, гравий
- отмечаются участки, которые лучше избегать по польским правилам движения
- видно текущее местоположение телефона
- можно выбрать две точки и получить маршрут для роликов
- алгоритм предлагает три варианта: лучшее покрытие, самый быстрый, самый безопасный

## Что уже реализовано

- `flutter_map` + OpenStreetMap для отображения карты
- загрузка дорог из OpenStreetMap через Overpass API
- маршрутизация через OSRM (`foot` profile как ближайший публичный вариант для роликов)
- эвристика качества покрытия и безопасности
- отдельная доменная логика в `lib/src/domain/skate_rules.dart`
- базовые тесты для классификации покрытий и оценки маршрута

## Важное ограничение

В текущей среде не установлен `flutter`, поэтому нативные каталоги `ios/` и `android/`
не сгенерированы автоматически. Код приложения и структура Flutter-проекта уже подготовлены,
но после установки Flutter нужно выполнить генерацию платформенных файлов.

## Как запустить после установки Flutter

1. Установить Flutter SDK.
2. Перейти в папку проекта:

```bash
cd warsaw_skate_app
```

3. Сгенерировать платформы, не перезаписывая `lib/` и `pubspec.yaml`:

```bash
flutter create --platforms=ios,android .
```

4. Загрузить зависимости:

```bash
flutter pub get
```

5. Запустить:

```bash
flutter run
```

## iOS и Android permissions

После генерации платформ нужно добавить разрешения:

- iOS: `NSLocationWhenInUseUsageDescription` в `ios/Runner/Info.plist`
- Android: `ACCESS_FINE_LOCATION` и `ACCESS_COARSE_LOCATION` в `android/app/src/main/AndroidManifest.xml`

## Юридическая логика

Маркировка "избегать по закону" в текущем прототипе работает как эвристика на основе OSM-тегов.
Сейчас логика исходит из того, что ролики относятся к `urządzenie wspomagające ruch`, и приложение
считает базово допустимыми:

- `footway`
- `pedestrian`
- `path`
- `cycleway`
- `living_street`

Как зоны, которых стоит избегать, MVP помечает:

- сегменты с `foot=no` или `access=no`
- магистральные дороги
- обычную проезжую часть, когда в данных есть отдельный `sidewalk`
- прочие дорожные типы, которые не выглядят как инфраструктура для пешего/роллерного движения

Это именно MVP-эвристика. Перед реальным релизом лучше вынести правовые правила в отдельный модуль и
сверить с актуальным польским правом и локальной юридической проверкой.
